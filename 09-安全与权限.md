# 09 安全与权限

> 一句话：**SSO 统一登录 + 三组件 RBAC + 客户端无长期凭证 + 全链路审计**。
> 
> ⚠️ **当前 demo 版本暂未启用加密**：以下涉及密钥管理、客户端加密等内容标注为"待实现"，当前 demo 的 decrypt-proxy 仅做透传。

## 1. 身份认证（SSO）

- 推荐用 **Gitea 自带 OAuth2/OIDC Provider**（小团队最省事，账号即 Git 账号）。
- 大一点可放 **Authentik / Keycloak**。
- 接入对象：
  - Jenkins（OIDC plugin）
  - Nexus（OIDC plugin / SAML）
  - Sentry（OIDC / OAuth）
  - MinIO（OIDC）
  - Filebrowser（OIDC）
  - **decrypt-proxy**（OIDC，必须）
  - Caddy（forward_auth 作为兜底）

## 2. 角色（RBAC，跨组件对齐）

| 角色 | Git | Jenkins | Nexus | Sentry | MinIO | decrypt-proxy |
| --- | --- | --- | --- | --- | --- | --- |
| `developer`        | 写自己仓库     | 跑 dev/staging job | 拉所有，写 dev | 读自己 project | 读 dev/staging | 搜索 + 下载 dev/staging |
| `qa`               | 只读           | 跑 staging job、审批提测 | 拉所有，写 staging | 读所有 | 读 staging | 搜索 + 下载 staging |
| `release-manager`  | 写 release/*   | 审批 release | 写 release（ALLOW_ONCE） | 读所有 | 读所有 | 搜索 + 下载 release |
| `sre / admin`      | admin          | admin | admin | admin | admin | admin（管升级/审计） |
| `external-customer`| —              | — | 仅拉 release | — | — | — |

> 角色挂在 SSO 组里（`group: developers/qa/release-managers/sre`），各组件读 group claim 完成授权，**只在一个地方加人**。

## 3. 网络分区

```
[公司内网]                              [DMZ / 公网?]
   开发者浏览器 ────► Caddy (TLS) ───► 各服务容器内部
                       │
                       └── OIDC → Gitea

[客户终端 (外网)]
   APP ───► token.intra (TLS) ──► upload-token-service
        ──► s3.intra (TLS, presigned PUT) ──► MinIO
```

- 内部服务（Jenkins、Nexus、Sentry、Filebrowser、MinIO Console、decrypt-proxy）**只对内网**或 VPN 暴露。
- 对外暴露的只有 `token.intra` 和 `s3.intra`（且 s3.intra 只接受带签名的 PUT/GET）。
- MinIO 的 admin 接口 9001 绝对不开公网；S3 接口 9000 通过 Caddy 反代 + WAF 规则。

## 4. 凭证管理

| 凭证 | 哪里存 | 谁能拿 |
| --- | --- | --- |
| MinIO 后台 root | `.env` / Vault | sre |
| MinIO 服务账号（app-uploader/proxy-reader） | proxy env / 容器环境变量 | 对应进程 |
| Nexus CI 账号 | Jenkins credentials | Jenkins |
| Sentry token | Jenkins credentials | Jenkins |
| Gitea webhook secret | Jenkins credentials | Jenkins |
| 代码签名证书（Win/Apple） | macOS Keychain / Windows cert store on build agent | 仅该 agent |

> ⚠️ 当前 demo 中 MinIO 凭证直接写在 compose.yaml 环境变量和 `.env` 中，生产环境应放到 Vault 或至少用 docker secrets。

### 4.1 客户端安全（待实现：加密链路启用后）

以下内容为未来加密方案设计，当前 demo 暂不涉及：

- **不下发 S3 access key**：用 presigned URL，TTL 1 小时。
- **master key 管理**：仅存在于 decrypt-proxy 服务器内存（从 `keyring.json` 读取），客户端不持密钥。
- **device_id 防伪造**：activate 时校验 app 签名 + 包名 + 防 root/debug。
- **完整性**：上传 sha256 双向校验，objects bucket 开 versioning。

## 5. 服务端硬化

- portal 主机：
  - 关闭 root SSH，仅 key 登录。
  - ufw / firewalld 仅放 22/80/443。
  - docker rootless 或限制 capabilities。
  - 自动化 patch：unattended-upgrades（Debian）/ dnf-automatic（RHEL）。
- 容器：
  - 自有 `sdk-docker` registry 镜像每周扫描（trivy）。
  - 容器 read-only filesystem，写卷只挂数据目录。
- TLS：Caddy 自动续签，最低 TLS1.2；内部服务也强制 TLS（mTLS 可选 between proxy ↔ MinIO）。

## 6. 审计 / 日志

| 系统 | 关键事件 |
| --- | --- |
| Jenkins         | login, job start/finish, input approval（含 submitter） |
| Nexus           | upload/download/delete（带 user/repo） |
| Sentry          | release create, debug-file upload, user invite |
| MinIO           | bucket notification → `audit/` topic，包含 GET/PUT/DELETE |
| decrypt-proxy   | login, search, download（JSONL） |
| Caddy           | access log（敏感字段不记） |

集中归集：把所有 json 日志通过 Vector / Fluent Bit 推到 Loki / OpenSearch，前置 Grafana 看；保留 1 年。

## 7. 合规与法务最小要求

- 所有 release 的 SDK 包内置 LICENSE + SBOM（`syft -o cyclonedx-json`）。
- 用户协议中说明日志/dump 上传内容、范围、保留期、删除请求接收点。
- GDPR/个保法配套：在 decrypt-proxy 提供按 `device_id` 删除（MinIO 批量删 + catalog 标记 erased）。

## 8. 灾备演练

每季度跑一次：
1. portal 主机模拟宕机，从异机房恢复。
2. 模拟一次 release 包被误删 → 从备份恢复，验证 sha 与 sentry release 关联仍正常。

> ⚠️ 加密方案中的灾备项（如 keyring.json 丢失恢复、Shamir secret sharing 等）留待后续版本实现。

## 9. 一句话总结

> **凭证最小化、SSO 统一、RBAC 对齐、链路全审计**。中小团队只用三件事就够：Vault 管秘、Gitea 管身份、Loki 管日志。加密链路留待后续版本补齐。
