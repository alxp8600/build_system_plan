# 02 技术选型（每个能力点都给候选 + 推荐）

按"轻量版 / 标准版 / 扩展版"列出，团队按预算和规模挑。**默认推荐 = 标准版**。

## 1. 源码托管 + Webhook

| 方案 | 优势 | 劣势 |
| --- | --- | --- |
| GitHub / GitLab.com SaaS | 0 运维 | 大文件、私有性、合规 |
| **Gitea**（self-host，单二进制/镜像） | 极轻，2 分钟起；webhook、PR、issue 都有 | 高级 CI 不及 GitLab |
| GitLab CE（self-host） | 一体化 CI/CD/Registry/Pages | 资源消耗大（>=4G） |

**推荐**：已有 GitHub/GitLab 直接复用；否则上 **Gitea**。

## 2. 编译调度（核心问题 1）

| 方案 | 是否合适 |
| --- | --- |
| **Jenkins（推荐）** | ✅ 老牌、插件最全、macOS/iOS agent 最成熟、Shared Library 复用强、支持 input 审批。 |
| GitLab CI | 适合已经全 GitLab 的团队；macOS Runner 自建较繁。 |
| Gitea Actions / GitHub Actions（self-host runner） | 上手快，但多平台矩阵/多 stage/审批不如 Jenkins 灵活。 |
| Drone / Woodpecker / Buildkite | 轻量但生态/插件少，iOS 支持弱。 |
| TeamCity | 商业，免费版 100 个 build config 够用，但插件不如 Jenkins 多。 |

**结论：选 Jenkins。** 理由：
- macOS agent + Xcode 全套教程最多；
- `Pipeline` + `Shared Library` 可以让 N 个 SDK 仓库共用一份模板；
- `input` step 天然支持"提测/发布"双人审批；
- 与 Gitea/GitHub/GitLab webhook 完全解耦。

## 3. SDK 二进制包仓库（核心问题 2 之一）

| 方案 | 适合什么 |
| --- | --- |
| **Sonatype Nexus 3 OSS（推荐）** | 同时支持 Raw（任意文件）/Maven（Android aar）/CocoaPods/npm/Docker；目录式浏览；REST API；账号 + 角色控制；社区版免费。 |
| JFrog Artifactory OSS | 功能类似，OSS 版能力被砍较多。 |
| 直接放 MinIO bucket | 最简，但没有 Maven/CocoaPods 协议、没有版本元数据 UI。 |
| Verdaccio (npm)、私有 Maven (Reposilite)、自建 CocoaPods spec repo | 单协议方案，需要多仓库拼。 |

**结论：Nexus 3 OSS。** 一个实例同时托管 Android Maven、iOS Cocoapods、Windows/Linux raw、Docker 镜像。

## 4. 符号表 + Crash 聚合（核心问题 2 之一）

| 方案 | 评价 |
| --- | --- |
| **Sentry（self-host docker-compose）（推荐）** | 原生支持 Breakpad/Native dump + iOS dSYM + Android ProGuard mapping + Windows PDB；自动符号化；issue 聚合 + 飞书/钉钉/邮件通知；UI 完善。 |
| Firebase Crashlytics | 易用但日志/dump 必须走 Google，国内访问难、隐私合规问题。 |
| Bugsnag / Backtrace.io | 商业，贵。 |
| 仅用 breakpad `minidump_stackwalk` + 自己脚本 | 免费但没有聚合 UI，只适合"应急" |

**结论：Sentry self-host。** 符号表用它的"Debug Files"功能管理，提测/发布通道作为 `release` 字段写入，自动关联同版本 crash。

## 5. 日志/Dump 原始文件存储（核心问题 3）

| 方案 | 评价 |
| --- | --- |
| **MinIO（推荐）** | S3 兼容、单二进制起、Web Console、生命周期策略、事件通知、纠删码、跨机房复制；客户端用任何 S3 SDK。 |
| 阿里云/腾讯云/AWS S3 | 0 运维；按量计费；合规需自查。 |
| 本地 NFS + Nginx autoindex | 最便宜，缺 ACL/生命周期/事件通知，浏览体验差。 |
| Ceph / SeaweedFS | 适合 PB 级，对中小团队过重。 |

**结论：MinIO（单节点起，后续可扩 4 节点 EC）**，按需挂云对象存储做异地备份。

## 6. 客户端怎么上传（不要写自己的上传服务）

| 方案 | 评价 |
| --- | --- |
| **MinIO Presigned URL（推荐）** | 客户端不持久化任何 access key；上传/下载通通用临时 URL；MinIO 原生功能。 |
| 自己写 upload server | 又一个要维护的服务。 |
| 通过 CDN 边缘上传 | 量大才有必要。 |

**做法**：写一个 ~50 行的 FastAPI 服务 `upload-token-service`，鉴权（JWT/HMAC）通过后用 `boto3` 生成 5 分钟有效 presigned PUT URL 返回给客户端。

## 7. 网页查看/下载日志 dump（核心问题 4）

| 方案 | 评价 |
| --- | --- |
| **MinIO Console（推荐自带 0 成本）** | 同时是浏览器；树形目录；下载；策略管理；登录与 MinIO 账号体系一致。 |
| **Filebrowser** | 更适合"团队多人共用一个目录"心智，支持账号、分享链接。 |
| Nextcloud | 太重。 |
| 自己写前端 | 没必要，先上面两个用着，等真的需要再写。 |

**做法**：MinIO Console 给 admin 用；Filebrowser 给开发/测试用（只读）；解密 = 走旁路 `decrypt-proxy`（见 07 章）。

## 8. 加密 + 解密下载 = 自写"decrypt-proxy"（唯一必须自研）

- **不要**让 SDK 上明文。日志压缩前先 AES-GCM 加密；密钥派发由 `upload-token-service` 一并完成。
- **不要**把"解密"做成网页里的 JS（密钥下发到浏览器=泄漏）。
- **正确做法**：写一个 FastAPI 服务，浏览器点击下载时反代过来：
  ```
  GET /d/{bucket}/{key}?fmt=plain
  → boto3.get_object(...)
  → AES-GCM 解密（密钥从内存 keyring，按 key_id 路由）
  → zstd 解压
  → StreamingResponse 返回，Content-Disposition: attachment; filename=…log
  ```
- 200 行以内，骨架见 `scripts-stub/decrypt_proxy.py`。

## 9. 反向代理 / TLS / 鉴权

| 方案 | 评价 |
| --- | --- |
| **Caddy 2（推荐）** | 配置极简（Caddyfile）；自动 Let's Encrypt；内置 basicauth/forward_auth/OIDC（caddy-security）。 |
| Nginx + acme.sh | 经典但配置多。 |
| Traefik | 适合容器化场景。 |

## 10. 通知

- Jenkins 用 `httpRequest` 直接 POST 到 飞书/钉钉/企业微信 webhook（不用插件）。
- Sentry 用内置 webhook 集成飞书/钉钉。
- MinIO 用 bucket event → webhook（可选，用于事件审计）。

## 11. SSO / 账号

- 小团队：所有组件开 LDAP 接入 **OpenLDAP** 或直接复用 Gitea 用户（Gitea 自带 OAuth2 Provider）。
- 更大团队：放 Authentik / Keycloak 做 OIDC，所有组件作为 client 接入。

## 12. 备份

- MinIO → `mc mirror` 到云 OSS / 异机房 MinIO（定时 cron）。
- Nexus → blobstore 定时打包到 MinIO/OSS。
- Sentry → 内置 PostgreSQL 定时 `pg_dump` 到 MinIO。
- Jenkins → `JENKINS_HOME` 整目录定时备份。

## 选型决策一句话

> 用 **Jenkins** 调度，用 **Nexus** 管 SDK 包，用 **Sentry** 管符号表+Crash，用 **MinIO** 存日志/dump 原文，用 **Filebrowser/MinIO Console** 看，用 ~200 行的 **decrypt-proxy** 在网页上一键得到明文。Caddy 做入口。需要写代码的只剩这个 proxy + 上传 token + 一份 Jenkins Shared Library。
