## 数据流（4 条主线）

### A. 出包流（git → 安装包）

1. 开发 `git push` 到 dev/test/release 分支或打 tag。
2. Gitea/GitHub 通过 webhook 通知 Jenkins。
3. Jenkins 加载共享库 `sdk-pipeline`，按平台 `parallel` 分发到 agents：
   - Android：`./gradlew assembleRelease` → `.aar`
   - iOS/macOS：`xcodebuild` → `.xcframework`
   - Windows：`cmake --build` → `.dll/.lib/.pdb`
   - Linux：`cmake --build` → `.so`
4. 构建完成后 agent 调 `nexus-cli` / `curl` 把产物 PUT 到 Nexus 对应仓库 `raw-dev / raw-staging / raw-release`，路径里带 `product/platform/version-build/`。
5. 同时 agent 调 `sentry-cli upload-dif` 把符号表（`.so`/`.dSYM`/`.pdb`/`mapping.txt`）传 Sentry。
6. Jenkins 发飞书/钉钉机器人通知（下载链接 + 构建摘要）。

### B. 提测/发布版本流转（dev → staging → release）

- **不重新编译**，只在 Nexus 里"挪/拷"二进制：
  - `raw-staging` ← Jenkins 手动 stage（"提测"按钮，需输入版本号/Jira 单号）。
  - `raw-release` ← 同样手动 stage，但加 `input` 双人审批。
- 通过 Nexus REST API 的 `copy`（或用 cron 把目录硬链接），保证字节一致。
- 版本号规范：`MAJOR.MINOR.PATCH[-rc.N|-dev.<shortsha>]`，每次升通道只是改 metadata + 加一个 tag。

### C. 客户端上传流（SDK → MinIO）

1. SDK 启动从配置中心拿到 **`upload-token-service`** 的地址（这是个 ~50 行的 Python 服务，下面 06 章会讲；存在的唯一目的：发 MinIO presigned PUT URL，避免把 access key 嵌进客户端）。
2. SDK 用本机对称密钥（来自配置中心，按 `key_id` 轮换）将日志/dump 做 **AES-GCM + zstd** 包成 `.enc`。
3. SDK 用 presigned URL `PUT` 到 MinIO，对象 key：
   - 日志：`s3://sdk-logs/{product}/{yyyymmdd}/{device_id}/{session}-{seq}.log.zst.enc`
   - dump：`s3://sdk-dumps/{product}/{sig}/{device_id}-{ts}.dmp.enc`
4. MinIO 事件通知（webhook / NATS）可触发 `decrypt-proxy` 异步生成"明文副本"或建立索引（可选）。

### D. 排障流（人 → 网页 → 明文）

1. 工程师打开 `https://sdk-portal/`（Caddy 反代）。
2. 默认页是 **Filebrowser**：登录后看到 `logs/` 和 `dumps/` 两个挂载的 MinIO bucket（用 rclone mount 或直接展示 MinIO）。
3. 点某个 `xxx.log.zst.enc`，链接其实是 `https://sdk-portal/d/<bucket>/<key>` → 转给 **decrypt-proxy** → 它流式读 MinIO → AES-GCM 解密 → zstd 解压 → 直接 `Content-Disposition: attachment; filename=xxx.log` 返回明文。
4. dump 同理；并且 decrypt-proxy 提供 `/symbolize?key=...` → 调用本机 `minidump_stackwalk` + 从 Sentry 下载的 symbols → 返回堆栈文本。
5. Crash 聚合直接看 **Sentry**（与原始 dump 互补：Sentry 看趋势/同类聚合，MinIO 看完整 dump+log 上下文）。

## 各组件分工 (One-liner)

| 组件 | 唯一职责 | 不做什么 |
| --- | --- | --- |
| Gitea/GitHub | 源码托管 + Webhook | 不做产物存储 |
| Jenkins | 编译调度 + 通道流转审批 | 不做长期产物存储 |
| Nexus 3 | SDK 二进制包仓库 (raw-dev/staging/release) | 不存日志/dump |
| Sentry | 符号表 + Crash 聚合 + Issue 跟进 | 不存完整 dump 原文 |
| MinIO | 日志/Dump 原始文件（加密密文） | 不存 SDK 包 |
| Filebrowser / MinIO Console | 网页浏览/下载密文 | 不解密 |
| decrypt-proxy（自写, ~200 行） | 流式解密 + 解压 + 符号化 | 不做存储/不做账号 |
| Caddy/Nginx | 统一入口 + TLS + OIDC/Basic | 不存数据 |
| upload-token-service（自写, ~50 行） | 给客户端发 MinIO presigned URL + 派发当前对称密钥的 key_id | 不接收数据 |
| Jenkins Shared Library（自写, Groovy） | 把"checkout→build→pack→push-nexus→push-sentry→notify"封成 1 个调用 | 不做业务编译 |

> **自研代码**全部加起来 ≈ 1 个 Python 微服务（decrypt-proxy + upload-token 合一也行）+ 1 套 Groovy 共享库 + 几个 shell 胶水脚本。其余 100% 现成镜像。
