# 11 需要自己写的脚本清单

> 一句话：**整个系统真正需要自研的代码不超过 1200 行**，全部集中在 4 个地方。其余全是配置 + 开源组件粘合。
>
> ⚠️ **当前 demo 版本暂未启用加解密**：decrypt-proxy 目前仅做透传下载，`client-tools` 不再包含加密/解密/密钥生成代码。加解密链路留待后续版本补齐。

| 序号 | 资产 | 语言 / 形式 | 估算行数 | 作用 |

### 2.7 `cdc/projects/windows/`（CDC Windows 构建脚本，实际已实现）

与 Shared Library 的 bash 模板不同，这是 CDC 项目 Windows 平台的实际出包+上传脚本栈：

```
cdc/projects/windows/
├─ build.bat              # cmake 配置 + 编译（Debug/Release）
├─ package.bat            # 提取 cdc_version.h 版本号 + 打 zip（带秒级时间戳）
├─ upload.bat             # MinIO 上传 wrapper：解析参数、读取 _last_zip.txt、调用 upload.ps1
├─ upload.ps1             # 核心：SigV4 签名 HEAD 检查 + PUT 上传 + GET 验证 + 可选清理本地文件
├─ .env                   # 本地测试用 AK/SK（Jenkins 上由凭据绑定注入）
├─ create_test_zip.ps1    # 本地冒烟测试：生成假 zip 验证上传链路
└─ package/
    ├─ _last_zip.txt      # package.bat 写入的最新 zip 文件名（upload.bat 通过此文件传递文件名）
    └─ cdc-win-*.zip      # 构建产物
```

**上传 MinIO 树形路径**（`upload.ps1` 自动解析）：

```
s3://sdk-packets/cdc/{branch}/windows/{version}/{date}/{zip_name}
                 示例: cdc/develop/windows/1.0.0/20260616/cdc-win-1.0.0-develop.51+e2de945-20260616-183702.zip
```

- **分支提取**：从版本号 `*-develop.*` → `develop`，`*-hotfix.*` → `hotfix`，`*-rc.*` → `staging`，纯数字 → `release`
- **日期精度**：从 zip 文件名时间戳 `YYYYMMDD` 提取，精度到天
- **幂等**：上传前 HEAD 检查，已存在则跳过

**Jenkins Freestyle Job 调用链**：

```cmd
rem Step 1: Build
call projects\windows\build.bat Release

rem Step 2: Package → 写入 package\_last_zip.txt
call projects\windows\package.bat Release

rem Step 3: Upload → 读取 _last_zip.txt → HEAD + PUT + 验证
call projects\windows\upload.bat "" "http://minio.internal:9000" "%MINIO_JENKINS_AK%" "%MINIO_JENKINS_SK%"
```

## 3. 不需要自己写的（开源接管）

- 包托管 → **Nexus**
- 符号化 + crash 聚合 → **Sentry**
- 对象存储 + 浏览 + lifecycle → **MinIO** (+ Console + Filebrowser)
- 调度 + 审批 + agent → **Jenkins**
- 反代 + TLS + 基础 auth → **Caddy**
- SSO → **Gitea OAuth2 Provider**
- 监控 → **Prometheus + Grafana + Loki + Vector**

## 4. 维护成本预估

| 角色 | 上线后周维护工时 |
| --- | --- |
| sre（兼） | 2~4 小时（看告警 / 升组件） |
| 后端开发（兼） | 1~2 小时（迭代 decrypt-proxy / upload-token） |
| Jenkins 流水线维护 | 1 小时（新 SDK 接入或调通道） |

> 用开源把 80% 的代码消解掉，自研聚焦在"胶水粘合"这一件事上 —— 这是中小团队最舒服的形状。

## 5. 一句话总结

> **真正的自研只剩 4 个目录**：`decrypt-proxy`、`upload-token`、`client-tools`、`jenkins-shared-lib`。所有骨架在 `scripts-stub/` 和 `demo/` 中提供，照着改一改就能上线。当前版本不做加解密，代码量进一步减少至 ~1300 行。
