# 11 需要自己写的脚本清单

> 一句话：**整个系统真正需要自研的代码不超过 1500 行**，全部集中在 4 个地方。其余全是配置 + 开源组件粘合。

## 1. 总览

| 序号 | 资产 | 语言 / 形式 | 估算行数 | 作用 |
| --- | --- | --- | --- | --- |
| 1 | `decrypt-proxy/`        | Python (FastAPI)            | ~400 | 网页端搜索 + 解密 + 流式下载 + 审计 |
| 2 | `upload-token/`         | Python (FastAPI)            | ~150 | 客户端换 presigned URL + 写 catalog |
| 3 | `client-tools/`         | Python + C++ 头文件示例     | ~250 | 上传/解密 CLI + SDK 端示例代码 |
| 4 | `jenkins-shared-lib/`   | Groovy + bash               | ~400 | `sdkPipeline()` + 通用 `build/package/publish` 脚本 |
| 5 | `infra/`                | Caddyfile / compose / sql / mc ilm json | ~300 | 一次性配置 |
| 6 | `backup/`               | bash + systemd unit         | ~100 | 定时备份 4 个组件 |
|   | **合计**               |                              | **~1600** | |

## 3. 不需要自己写的（开源接管）

- 包托管 → **Nexus**
- 符号化 + crash 聚合 → **Sentry**
- 对象存储 + 浏览 + lifecycle → **MinIO** (+ Console + Filebrowser)
- 调度 + 审批 + agent → **Jenkins**
- 反代 + TLS + 基础 auth → **Caddy**
- SSO → **Gitea OAuth2 Provider**
- 监控 → **Prometheus + Grafana + Loki + Vector**

## 4. 维护成本预估

| 角色 | 上线后周维护工时 |
| --- | --- |
| sre（兼） | 2~4 小时（看告警 / 升组件 / 转 key） |
| 后端开发（兼） | 1~2 小时（迭代 decrypt-proxy / upload-token） |
| Jenkins 流水线维护 | 1 小时（新 SDK 接入或调通道） |

> 用开源把 80% 的代码消解掉，自研聚焦在"加密链路"这一件事上 —— 这是中小团队最舒服的形状。

## 5. 一句话总结

> **真正的自研只剩 4 个目录**：`decrypt-proxy`、`upload-token`、`client-tools`、`jenkins-shared-lib`。所有这些骨架都在 `scripts-stub/` 提供，照着改一改就能上线。
