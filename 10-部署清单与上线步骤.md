# 10 部署清单与上线步骤

> 一句话：**所有组件 docker-compose 一把起，2 个工作日完成基础环境**。

## 1. 总 compose 文件（节选，完整骨架见 `scripts-stub/compose.yaml`）

```yaml
# /srv/sdk-portal/compose.yaml
name: sdk-portal

services:
  caddy:
    image: caddy:2
    restart: unless-stopped
    ports: ["80:80", "443:443"]
    volumes:
      - ./caddy/Caddyfile:/etc/caddy/Caddyfile:ro
      - ./caddy/data:/data
      - ./caddy/config:/config

  gitea:
    image: gitea/gitea:1
    restart: unless-stopped
    volumes: ["./gitea/data:/data"]
    environment:
      - GITEA__server__ROOT_URL=https://git.intra/
      - GITEA__server__SSH_DOMAIN=git.intra
      - GITEA__oauth2__ENABLE=true
    networks: [portal]

  jenkins:
    image: jenkins/jenkins:lts-jdk17
    restart: unless-stopped
    volumes:
      - ./jenkins/home:/var/jenkins_home
      - ./jenkins/casc:/var/jenkins_home/casc:ro
    environment:
      - CASC_JENKINS_CONFIG=/var/jenkins_home/casc/jenkins.yaml
    networks: [portal]

  nexus:
    image: sonatype/nexus3:latest
    restart: unless-stopped
    volumes: ["./nexus/data:/nexus-data"]
    networks: [portal]
    deploy:
      resources:
        limits: { memory: 4g }

  minio:
    image: minio/minio:latest
    restart: unless-stopped
    command: server /data --console-address ":9001"
    volumes: ["./minio/data:/data"]
    env_file: .env
    networks: [portal]

  sentry:
    extends:
      file: ./sentry/docker-compose.yml      # 来自 getsentry/self-hosted
      service: web
    # ... 略, 直接复用官方 self-hosted compose

  filebrowser:
    image: filebrowser/filebrowser:latest
    restart: unless-stopped
    volumes:
      - ./filebrowser/database.db:/database.db
      - ./filebrowser/config.json:/.filebrowser.json
      - ./minio/data:/srv:ro     # 只读挂载, 浏览不影响 minio
    networks: [portal]

  decrypt-proxy:
    build: ./decrypt-proxy
    restart: unless-stopped
    volumes:
      - ./decrypt-proxy/keyring.json:/keyring.json:ro
    env_file: .env
    networks: [portal]

  upload-token:
    build: ./upload-token
    restart: unless-stopped
    env_file: .env
    networks: [portal]

  postgres-catalog:
    image: postgres:16
    restart: unless-stopped
    volumes: ["./postgres-catalog:/var/lib/postgresql/data"]
    env_file: .env
    networks: [portal]

networks:
  portal:
```

## 3. 上线步骤（按顺序）

### Day 1：基础

1. 准备 portal 主机，挂数据盘 `/srv` (RAID10)，安装 docker / compose v2 / git。
2. 配 DNS：把上面九个域名都 A 记录到 portal 主机。
3. 克隆 `sdk-portal` 配置仓库到 `/srv/sdk-portal`。
4. 填 `.env`（MinIO root、Postgres、JWT secret、Sentry secret-key、Gitea OIDC client、飞书 webhook 等）。
5. 生成密钥：
   ```
   openssl rand -base64 32 > /srv/sdk-portal/decrypt-proxy/keyring/kr-$(date +%Y-%m).key
   # 编辑 keyring.json 把 base64 内容写入
   chmod 600 /srv/sdk-portal/decrypt-proxy/keyring.json
   ```
6. `docker compose up -d caddy gitea`，初始化 Gitea OIDC Provider。
7. `docker compose up -d minio`，进 console 建 4 个 bucket、配置 lifecycle、建服务账号 `jenkins-uploader / proxy-reader / app-uploader`。
8. `docker compose up -d nexus`，按 03 章建 8 个仓库，配 Cleanup Policy，建 CI 账号。
9. `docker compose up -d sentry`（首次按官方 `install.sh` 走完），建 4 个 project，建 release-manager / dev role。

### Day 2：CI 与自研服务

10. `docker compose up -d jenkins`，通过 JCasC 自动应用：用户角色、credentials、Shared Library、Gitea 连接器。
11. 给 Jenkins 接入 macOS / Windows / Linux agent（各自跑 `swarm-client.jar` 或 SSH agent）。
12. 把 SDK 仓库的 Gitea webhook 指到 Jenkins，触发一次 dev 编译 → 自检通过。
13. `docker compose up -d postgres-catalog upload-token decrypt-proxy filebrowser`，初始化 catalog DB schema（`scripts-stub/sql/init.sql`）。
14. 用 `client-tools/upload_test.py` 跑一遍 "上传 → ack → web 列表 → 解密下载 → sha 校验"。
15. 加监控：`docker compose up -d prometheus grafana loki vector`，导入 Grafana dashboard JSON。
16. 配备份 systemd timer：`backup-nexus.timer`, `backup-sentry-pg.timer`, `backup-catalog.timer`, `mirror-minio.service`。
17. **演练**：手动停 portal，从备份恢复到一台空机；验证恢复时间 < 2 小时。

### Day 3+：渐进上线

18. 把第一个 SDK 仓库正式接入（Jenkinsfile + ci/scripts），跑通 dev → staging → release 三通道。
19. 客户端 SDK 接入 `upload-token-service` & MinIO 直传；先 dev 通道灰度。
20. 设置 Sentry 告警规则 + 飞书集成。
21. 培训：开发 30min、QA 30min、release-manager 30min（含 GPG tag 演示）。
22. 公布 wiki，进入"运营态"。

## 4. 验收清单（上线前最后一关）

- [ ] 任意 SDK 仓库 `git push` → Jenkins 自动出 dev 包，飞书有通知含 Nexus 下载链接。
- [ ] 打 `vX.Y.Z-rc.1` tag → staging 通道 `input` 审批生效，未审批不写 Nexus。
- [ ] 打 `vX.Y.Z` 签名 tag → release 通道二次审批，写 `sdk-raw-release`；尝试重复 publish 同版本号必失败（`ALLOW_ONCE`）。
- [ ] Sentry 中能看到本次构建的 release，且符号表能将 demo crash 调栈到源码行。
- [ ] 客户端 demo 触发 crash → dump 加密上传 MinIO；catalog 出现条目；网页能"解密下载"，sha 与本地一致；用 dSYM/PDB 能调栈。
- [ ] 测试 lifecycle：把 dev 桶 tag 改 `keep=1d`，跑 minio `mc ilm` 测试，第二天对象消失，catalog 标 `deleted_at`。
- [ ] 备份恢复演练：portal 关机，从备份恢复到空机 → 所有 9 个域名能登录，最近 1 天数据无丢失。
- [ ] 关掉 decrypt-proxy，确认密文仍可通过 Filebrowser / MinIO Console 下载（紧急通道可用）。
- [ ] 审计：解密下载、release 发布、keyring 改动事件能在 Loki/Grafana 中查到。

## 5. 回滚策略

- **Jenkins Shared Library**：放独立仓库，按 tag 引入（`@Library('sdk-shared-lib@v1.4')`），出问题切回上一个 tag。
- **decrypt-proxy / upload-token**：镜像按版本号打 tag，compose `image: x:1.4.0`；回滚 = 改 tag + `up -d`。
- **Nexus release 仓**：永不回滚（不可变）；如线上版本有问题，发新 patch 版本，宣告作废老版本。
- **MinIO 数据**：开 versioning，误删可在 7 天 noncurrent 期内 `mc undo` 恢复。
- **Sentry**：升级前 `pg_dump`，回滚直接 restore。

## 6. 一句话总结

> **基础设施全部声明在 `/srv/sdk-portal/compose.yaml` + `.env`**，上线 = compose up，验收靠固定清单，回滚靠镜像 tag。两个工作日内可投入开发使用。
